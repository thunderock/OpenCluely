Move the Gemini-SPECIFIC network config out of unconditional global startup and INTO the Gemini
provider so it applies only when Gemini is the selected provider and disappears cleanly at removal.

In `src/services/providers/gemini.provider.js`, add a method `configureNetworkSession(electronSession)`
that contains the two hostname-guarded blocks relocated VERBATIM from `main.js` `setupNetworkConfiguration`
(current lines 294-310):
```
configureNetworkSession(ses) {
  if (!ses) return;
  ses.webRequest.onBeforeSendHeaders((details, callback) => {
    if (details.url.includes('generativelanguage.googleapis.com')) {
      details.requestHeaders['User-Agent'] = 'Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/122.0.6261.156 Safari/537.36';
    }
    callback({ requestHeaders: details.requestHeaders });
  });
  ses.setCertificateVerifyProc((request, callback) => {
    if (request.hostname === 'generativelanguage.googleapis.com') {
      callback(0);
    } else {
      callback(-2);
    }
  });
}
```
Keep the exact UA string and the exact callback codes (0 trust / -2 default) byte-identical. This is a
Gemini-only capability (NOT part of the 4-method interface) — a future Local/other provider simply
won't define it.

In `main.js`, edit ONLY the BODY of `setupNetworkConfiguration()` (keep the method name and its
invocation at line 223 intact — do NOT change any `llmService.*` call-site or the require at line 86).
Replace the two inline blocks with a gated delegation to the selected provider:
```
setupNetworkConfiguration() {
  const ses = session.defaultSession;
  const provider = require('./src/services/providers').getSelected();
  if (provider && typeof provider.configureNetworkSession === 'function') {
    provider.configureNetworkSession(ses);
  }
  logger.debug('Network configuration applied for selected provider');
}
```
Because `getSelected()` is the hardcoded Gemini provider this phase, behavior is unchanged now; when
Gemini is removed in Phase 3 the selected provider won't implement `configureNetworkSession`, so the
bypass vanishes with no dead global code (SC3).
