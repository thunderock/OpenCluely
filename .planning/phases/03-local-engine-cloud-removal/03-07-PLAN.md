<task type="auto">
  <name>Task 1: Write the scriptable TTFT/memory smoke harness</name>
  <files>scripts/smoke-local.js</files>
  <action>
Write `scripts/smoke-local.js` implementing the RESEARCH Flag 1 "rough smoke" procedure. Put it in `scripts/` (NOT `test/*.test.js`) so `make run_tests` never runs it (it needs a real Ollama + the pulled model). Node CJS, no new deps (use `ollama` / global `fetch`).

The script:
1. Reads `config.get('llm.local')` for host + model.
2. **Warm** the model with one throwaway request (so `load_duration` ≈ 0; measure the resident case).
3. Builds a **representative prompt** simulating the bounded md-notes budget:
   - system = the General reply-suggester prompt + ~12,000 chars of filler md-context (the `context.maxChars` budget Phase 5 will use);
   - one downscaled screenshot PNG as a base64 `image_url` data URL, long edge ~1280px (generate a synthetic ~1280px PNG if no sample is provided, or accept a path arg);
   - a short user question.
4. Sends it **non-streaming** via `/api/chat` (ollama-js `chat` or `fetch` to `/api/chat`) and reads `prompt_eval_duration` (ns → the TTFT/prefill proxy), plus `eval_count`/`eval_duration` (decode rate). Also times **wall-clock to first streamed delta** with a streaming call.
5. Prints a compact report: prefill ms, wall-clock-first-delta ms, decode tok/s, and a reminder to run `ollama ps` and confirm `PROCESSOR` = 100% GPU + note `SIZE`, and to check Activity Monitor memory pressure green / swap ≈ 0.
6. Prints a clear PASS/FAIL against the lenient Phase-3 gates (RESEARCH Flag 1): TTFT ≤ ~3–4 s wall-clock (warm), model fully GPU-resident, no swap. Exits non-zero on the timing gate so it can be eyeballed but never blocks CI (it is not a test-glob file).

Fail gracefully with a clear message if Ollama is down or the model is missing (point the user at onboarding/settings).
  </action>
  <verify>
`node -c scripts/smoke-local.js` (syntax) passes; `npx eslint scripts/smoke-local.js` exits 0. With a real Ollama + model present, `node scripts/smoke-local.js` prints the prefill/first-delta/decode report and a PASS/FAIL line. `make run_tests` does NOT execute it (it is not `test/*.test.js`).
  </verify>
  <done>A reusable, dependency-free smoke harness exists that measures prefill (prompt_eval_duration) + wall-clock TTFT on a representative bounded-notes + one-image prompt and reports the Phase-3 rough-gate PASS/FAIL.</done>
