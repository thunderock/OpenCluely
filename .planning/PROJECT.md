# OpenCluely

## What This Is

OpenCluely is a stealth desktop AI overlay (invisible to screen-sharing on macOS/Windows) — a **private, always-on general copilot that automatically suggests what to say or do next.** It watches your screen and hears the conversation, and after each natural pause surfaces a relevant reply suggestion — for any call, meeting, or task, **not just interviews or coding.** This milestone transforms it from a cloud-Gemini, on-demand tool into a **local-first, always-on multimodal copilot**: on launch it self-starts local speech-to-text and model services, loads a folder of your own `.md` notes as standing context, then continuously listens and watches the screen and surfaces a suggested reply after each natural speech pause.

## Core Value

After any natural pause in what you're saying or hearing, a **relevant, streamed answer appears in the stealth overlay** — generated by a **local** model from your screen + speech + your own notes — fast and private. If everything else fails, this must work.

## Context

- **Existing architecture** (full map in `.planning/codebase/`): Electron app; `main.js` is a god-object `ApplicationController` delegating to CommonJS singleton managers/services (`src/managers/`, `src/services/`, `src/core/`); vanilla-JS renderers with no framework, bundler, or TypeScript; `preload.js` is the sole IPC bridge.
- **Gemini coupling**: `src/services/llm.service.js` (1655 lines) *is* Gemini — three request mechanisms (SDK, raw HTTPS, hand-rolled SSE) and `generativelanguage.googleapis.com` hardcoded in ~6 places, plus an unconditional TLS cert-verify bypass and User-Agent override in `main.js`. There is no provider interface to slot a local model into.
- **Structural blockers to the vision** (from `CONCERNS.md`): (1) no LLM provider abstraction; (2) Whisper spawns a fresh Python process per utterance — the #1 blocker to continuous listening; (3) screenshot capture is single-shot, full-res, no throttle/dedup — blocker to continuous watching; (4) no supervisor for any long-running local service; (5) no md-context injection mechanism (`prompt-loader.js`'s directory-scan-into-`Map` is the closest skeleton, but it loads one hardcoded file at startup).
- **Dead/fragile code to clean during the refactor**: `src/ui/chat-window.js` (dead, unloaded — streaming chunk events currently unconsumed); `src/services/fallback-capture.service.js` (0 bytes); dead `close-window`/`quit-app` handlers; dead `global.windowManager` calls; orphaned Gemini-config modal with a persistence bug; Tailwind configured but not build-wired; `marked` dependency never loaded. License is stated three different ways (package.json ISC vs LICENSE Apache-2.0 vs README MIT).
- **Security to address alongside**: remove the cert bypass; sanitize LLM output before `innerHTML`; re-apply `0600` on `.env` rewrite (or move to `safeStorage`); consider scoping the broad `get-settings` IPC surface.
- **Target hardware**: Apple Silicon Mac, 32 GB+ unified memory (primary dev + use machine).
- **Prior art / references**: `thunderock/forge` (`../forge`) for headless `claude`/`codex` CLI invocation *and* its macOS DMG release workflow; `openwhispr` for local-model-service download/startup UX.

## Constraints

- **Tech stack**: Stay on Electron + CommonJS + vanilla-JS — extend existing patterns, no rewrite to a framework/bundler/TS. Match established conventions (including the intentional `assests/` misspelling).
- **Latency**: Continuous per-pause suggestions must feel real-time → local model is the only thing on that path. Claude/Codex CLI agents are never invoked per-pause (cold-start + cloud latency + usage cost).
- **Privacy**: Primary path fully local/offline. The only cloud is the opt-in CLI-agent backup, which reuses existing terminal login (no managed API keys stored by the app).
- **Platform**: Primary target Apple Silicon macOS (32 GB+); keep Windows/Linux building; screen-capture invisibility remains macOS/Windows only.
- **Signing**: No Apple Developer certificate → macOS DMG ships unsigned; document the `xattr -cr` workaround.
- **Delivery** (personal-project conventions): ship a `Makefile` (`setup`/`setup-dev`/`run_tests`/`lint`); always write tests; type where the language allows; small atomic Conventional Commits; never auto-push or open PRs.
