## Gotchas
- 2026-07-17 ‚Äî Headless-boot teardown: `pkill -f "OpenCluely/node_modules/electron"` kills only the HELPER processes ‚Äî the MAIN process rewrites its argv via `process.title = "Terminal "` (stealth) so the -f pattern misses it, and the surviving instance then blocks the next boot via the single-instance lock (log stalls at ~12 lines). Kill the main pid too: `MAIN_PID=$(pgrep -f "OpenCluely/node_modules/electron" | head -1 | xargs -I{} ps -o ppid= -p {}); kill -9 $MAIN_PID`.
- 2026-07-14 (updated 2026-07-17) ‚Äî gsd-tools CLI can't parse this repo's NARRATIVE ROADMAP/STATE format ‚Äî `phase complete` / `state advance-plan` / `update-progress` silently no-op or HALF-work AND `phase complete` falsely reports `is_last_phase: true` (2026-07-17: it advanced STATE frontmatter/position but left the ROADMAP checkbox + Progress table untouched AND clobbered a freshly-written `stopped_at` back to stale text; `state advance-plan` also mis-incremented the plan counter). Mark phase/plan/requirement completion MANUALLY in ROADMAP.md + REQUIREMENTS.md, re-check `stopped_at` after running it; verify next-phase via ROADMAP, not the CLI's `is_last_phase`.
- 2026-07-14 ‚Äî Parallel GSD executors share one branch/worktree: a bare `git commit` after `git add <file>` commits the whole staged index and can sweep a sibling's staged files into your commit (mislabeling attribution ‚Äî see `510a7da` labeled feat(01-03) but containing 01-01 files). Use explicit-pathspec commits: `git commit -- <files>`.
